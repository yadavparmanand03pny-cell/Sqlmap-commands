🎯 Complete Bug Bounty Methodology

Recon se attack tak — poora flow, ek jagah, jitna zyada info utna useful

— Phase 1: Recon —

Subdomain Enumeration FIRST STEP

1. subfinder -d target.com + crt.sh — saare subdomains collect karo
2. httpx se check karo kaunse subdomains live hain
3. Old/unused subdomains note karo — inpe Subdomain Takeover try hoga

Tech Fingerprinting KNOW THE STACK

1. whatweb target.com ya Wappalyzer extension — CMS, framework, server pata karo
2. Server header, cookies naming pattern se backend language guess karo (PHP, Node, Java)
3. Version number mile to known CVE search karo us version ke liye

Port & Service Scanning HIDDEN ENTRY POINTS

1. nmap -sV -p- target.com — saare open ports aur services
2. Non-standard ports pe admin panel ya staging environment mil sakta hai
3. nmap --script vuln target.com — known service vulnerabilities check

URL & Parameter Collection SURFACE MAPPING

1. waybackurls + gau — purane aur current dono URLs nikalo
2. Har URL ke query parameters note karo — attack surface yahi banti hai
3. Burp Spider/Crawl chalao live navigation ke through — dynamic URLs bhi mile

Hidden Parameter Discovery OFTEN MISSED

1. arjun -u https://target.com/endpoint — hidden parameters brute-force karta hai
2. paramspider -d target.com — historical URLs se parameters extract karta hai
3. Jo parameter UI mein nahi dikhta lekin backend accept karta hai — wahi hidden bugs ka source hota hai

API & Endpoint Discovery HIDDEN SURFACE

1. Network tab (Burp/DevTools) se XHR/fetch calls capture karo
2. /api/`, `/v1/`, `/v2/ jaise paths bruteforce karo (ffuf/kiterunner)
3. Swagger/OpenAPI docs try karo: /swagger-ui.html, /api-docs

JS File Analysis OFTEN IGNORED

1. Saari .js files download karo, linkfinder se endpoints nikalo
2. Hardcoded API keys, internal URLs, comments dhundo

Directory / File Bruteforcing HIGH FINDING RATE

1. ffuf -u https://target.com/FUZZ -w wordlist.txt
2. Backup files specifically try karo: .env, .git, config.php.bak, .zip
3. intitle:"index of" Google dork se bhi open directories milte hain

Screenshotting (Visual Recon) SPEED UP TRIAGE

1. gowitness scan file -f subdomains.txt — actively maintained, Chrome headless use karta hai isliye modern JS-heavy sites bhi properly load hoti hain
2. Jaldi identify ho jaata hai kaunsa login page, admin panel, ya dev/staging environment hai
3. Backup option: aquatone — purana lekin quick setup, kabhi kabhi heavy JS sites pe screenshot blank aa sakta hai

Cloud Storage / S3 Buckets MISCONFIG GOLDMINE

1. Company naam se bucket names guess karo: target-backup, target-uploads, target-dev
2. s3scanner ya grayhatwarfare.com se public buckets search karo
3. Misconfigured bucket mein directly sensitive files/data mil sakta hai

WAF Detection BEFORE ATTACKING

1. wafw00f target.com — pehle pata karo WAF hai ya nahi
2. WAF pata hone se payload customize kar sakte ho bypass ke liye (encoding, case variation)

— Phase 2: Attack Checklist —

Subdomain Takeover TRY FIRST

Dangling CNAME dhundo (subdomain kisi service — S3, Heroku, GitHub Pages — ko point kar raha ho jo ab exist nahi karti)
Check tool: subzy ya nuclei -t subdomain-takeover

Authentication Testing HIGH IMPACT

Login/OTP/password-reset flow pe rate limiting hai ya nahi
Session token predictable hai kya, logout ke baad token invalidate ho raha hai kya
Password reset link mein Host header manipulate karke dekho

SSRF SERVER-SIDE

Jo bhi parameter URL/webhook/callback leta hai — usme internal IP daalo (169.254.169.254, localhost)
File upload/PDF generator/image-fetch features pe bhi try karo — inme chhupa SSRF milta hai

SQL Injection CLASSIC

Search, filter, sort, login parameters pe ', ", OR 1=1 try karo
Error na aaye to time-based blind try karo: ' AND SLEEP(5)--

IDOR / Access Control MOST COMMON

Har ID-based endpoint pe apna ID doosre ke ID se replace karo
Numeric hi nahi — UUID/hex ID bhi try karo, low-privilege account se high-privilege data access try karo

XSS CLIENT-SIDE

Search box, comment, profile bio, review fields mein payload daalo: <script>alert(1)</script>
Reflected na mile to Stored XSS check karo — payload save hoke doosri jagah render ho raha hai kya

CORS Misconfiguration HEADER-BASED

Request mein random Origin header daal ke dekho response mein reflect ho raha hai kya
Agar Access-Control-Allow-Credentials: true ke saath reflect ho raha hai — critical finding

— Common Vulnerabilities on Websites —

Quick Reference FREQUENTLY FOUND

XSS — search box, comment, profile bio fields mein
IDOR — invoice, order, profile ID-based endpoints
Open Redirect — login/logout ke redirect parameters mein
CORS Misconfig — Origin header blindly reflect ho raha ho
Rate Limiting Missing — OTP, login, coupon apply pe
Info Disclosure — API response mein extra fields (email, internal ID)
CSRF — state-changing action bina token verification ke
Clickjacking — X-Frame-Options header missing
Weak Password Policy — signup/reset pe koi complexity check nahi
Directory Listing — intitle:"index of" se khula folder mil jaana
Subdomain Takeover — dangling DNS record kisi deleted service ko point kare
Mass Assignment — request body mein extra field (role, isAdmin) accept ho jaana
